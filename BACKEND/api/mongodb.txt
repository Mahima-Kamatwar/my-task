MongoDB is a NoSQL database (Not Only SQL).
Unlike traditional relational databases (like MySQL or PostgreSQL), MongoDB stores data in flexible, JSON-like documents instead of rows and tables.

Each record is called a document, and documents are stored inside collections.

sql and NoSQL difference

| Type      | Meaning                                                                                                                                |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| **SQL**   | *Structured Query Language* — relational databases with fixed tables, rows, and columns.                                               |
| **NoSQL** | *Not Only SQL* — non-relational databases that store data in flexible formats like JSON, key-value, graph, or column-based structures. |


| Feature               | **SQL (Relational DB)**                                                 | **NoSQL (Non-Relational DB)**                                             |
| --------------------- | ----------------------------------------------------------------------  | ------------------------------------------------------------------------- |
| 🧩 **Data Structure** | Tables with rows and columns                                           | Documents, key-value pairs, graphs, or wide-columns                       |
| 📊 **Schema**         | Fixed schema (you must define columns before inserting data)           | Dynamic schema (you can change structure anytime)                         |
| 🔗 **Relationships**  | Uses foreign keys and JOINs to relate data across tables               | Usually avoids JOINs — data is often nested within documents              |
| ⚙️ **Scalability**    | Vertically scalable (add more power to one server)                     | Horizontally scalable (add more servers)                                  |
| 💬 **Query Language** | Structured Query Language (SQL)                                        | Varies — depends on the database (e.g., JSON-style queries in MongoDB)    |
| 💾 **Storage Format** | Rows and columns                                                       | JSON-like documents, key-value pairs, etc.                                |
| 🧮 **Examples**       | MySQL, PostgreSQL, Oracle, MS SQL Server                               | MongoDB, Cassandra, Firebase, DynamoDB                                    |
| 🔐 **Transactions**   | Strong ACID properties (Atomicity, Consistency, Isolation, Durability) | Often BASE model (Basically Available, Soft state, Eventually consistent) |
| ⚡ **Best For**       | Complex queries, structured data, consistent relationships             | Large-scale, fast-changing, unstructured or semi-structured data          |
